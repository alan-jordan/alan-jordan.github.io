Design to Web

9th March 2017

What a responsive site is, and why responsiveness is important.

A responsive site is one which has been designed to work on any device, regardless of its screen size. Code can change how the site looks based on the screen width. So a site may have the same code but it will look different on a mobile phone, a table, a smart tv and a desktop.

Responsive sites are important for a number of reasons:

  • More and more web browsers are mobile and that is trending upwards;
  • It is easier to maintain code for a single site rather than mobile and desktop version. It is also cheaper;
  • It doesn't look off a certain user because of their device that will make users happy, AND;
  • Happy users is generally a good thing.

What mobile first design is, and why it's important.

Mobile first design is the practice of designing a site with mobile users in mind first and then expanding the width to accomodate users with wider screens.

This practice gives a number of benefits. By paring down the experience to the absolute minium, it allows a 'snappier' site to be run on mobile devices. It then allows the desktop experience to be gradually improved as screens get wider and devices ultimately get more powerful.

What frameworks are, and their pros and cons.

A framework is a collection of code (CSS, HTML and Javascript in this case) which assists developers in building a site. Front-end frameworks assist in things like responsive design, the positioning of elements, fonts, normalising the CSS etc. Backend frameworks do things like standardise repeatable tasks. Things like connecting to a database don't require a whole new db_connect function to be written like in the bad old days. They basically help things get done faster by doing a lot of the grunt-work up front.

What a wireframe is and why we use it.

A wireframe is a very basic mock up of a web site. It is a dumbed down version of a site which is usually in black and white.

It allows you to look at how a user might interact with a page and contains the basic design hierarchy of a site. It also allows you to look at the site without any clutter like fonts, colour, pictures etc.
